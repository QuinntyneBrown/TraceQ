# TraceQ — L1 Requirements

> **Air-gapped requirements intelligence platform for Space & Defense.**
> Imports Windchill PLM CSV exports into a local vector database for semantic search, analysis, and reporting via a web dashboard.

| Constraint | Detail |
|---|---|
| Security | All processing local — zero cloud calls for embeddings, storage, or inference |
| Backend | C# / ASP.NET Core 8 |
| Frontend | Angular 17, Angular Material, Chart.js (ng2-charts), angular-gridster2 |
| Embeddings | ONNX Runtime with all-MiniLM-L6-v2 (384-dim, CPU-only) |
| Vector DB | Qdrant (local standalone binary, no Docker required) |
| Metadata DB | SQLite via Entity Framework Core |

---

## L1-1: CSV Import Pipeline

The system shall import requirements exported from Windchill PLM in CSV format, parsing standard Windchill fields (Number, Name, Description, Type, State, Priority, Owner, dates, module, parent, traceability links) and persisting them to the metadata store. The import shall support incremental updates (re-import without duplicating existing requirements) and provide detailed import status feedback.

## L1-2: Local Embedding Generation

The system shall generate vector embeddings for all imported requirements using a locally-executed ONNX model (all-MiniLM-L6-v2) with no network calls. Embedding generation shall run on CPU, support batch processing, and automatically embed new or modified requirements.

## L1-3: Vector Storage & Indexing

The system shall store requirement embeddings in a locally-hosted Qdrant vector database and maintain synchronization between the metadata store (SQLite) and the vector index. The vector store shall support efficient approximate nearest-neighbor search.

## L1-4: Semantic Search & Query

The system shall provide a semantic search API and UI that allows users to find requirements by natural-language query, find similar requirements to a selected requirement, and filter/facet results by Windchill metadata fields (Type, State, Priority, Module, Owner).

## L1-5: Reporting & Visualization Dashboard

The system shall provide a configurable dashboard with draggable/resizable widgets (via Gridstack) displaying requirement analytics using Chart.js, including distribution charts, traceability coverage, similarity clusters, and status breakdowns. Users shall be able to save and load dashboard layouts.

## L1-6: Web Application Shell

The system shall provide an Angular-based single-page application with Angular Material theming, responsive layout, navigation, and a consistent design system suitable for enterprise/defense use.

## L1-7: Security & Air-Gap Compliance

The system shall enforce that no data or processing leaves the local machine. All network-capable components shall be configured for localhost-only binding. The system shall provide audit logging for data import and query operations.

## L1-8: UI Design System

The system shall provide a comprehensive, responsive UI design covering all screen layouts (mobile 375px, tablet 768px, desktop 1440px), interactive component states (buttons, toggles), user feedback patterns (toast notifications, inline alerts), modal dialogs (confirmation, form, detail, destructive), and error/empty state presentations. The design shall follow a professional enterprise aesthetic suitable for Space & Defense, using a consistent dark-themed color system with high-contrast accessibility.
